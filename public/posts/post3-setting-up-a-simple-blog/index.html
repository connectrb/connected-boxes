    Post#3: Long Weekends and Digital Homes: Why I Swapped LinkedIn for Hugo

    April 5, 2026

    This long weekend, I finally scratched an itch I’ve had for years: starting a personal blog.

    For a long time, I relied on LinkedIn to share my thoughts. But LinkedIn has a “version control” problem. Recently, I tried to edit an image in a post and watched the entire layout implode with no way to recover the old version. That was the final straw. I needed a digital home I actually owned—a place where I could break things, fix them, and keep a history of it all.

    The Architect: Gemini and Hugo#

    A quick brainstorming session with Google Gemini (my current go-to after some time with Perplexity) pointed me toward Hugo and GitHub Pages. It’s a lightweight, fast, and rock-solid combination that stays out of the way so I can focus on writing.

    The Foundation: Debian#

    I handled the setup on my primary laptop running Debian. I’ve been a Debian loyalist for years, ever since I started using it for Open Media Vault (OMV) to manage my home storage and Docker needs. There’s something satisfying about building on a “universal operating system” that just works—keeping my precious documents and memories safe. (I’ll likely write more about my OMV setup in a future post).

    The Force Multiplier: OpenCode + OpenRouter#

    While the setup was straightforward, I wanted to supercharge the process. I integrated OpenCode with OpenRouter as the backend. This setup is like having a co-pilot for the boring stuff—fixing Hugo template errors or double-checking Markdown syntax.

    I’ve been experimenting with a few different “brains” on OpenRouter:

    • Gemini 3 Flash: My daily driver—fast and reliable.
    • DeepSeek V3.2: Great for budget-friendly tasks, though a bit slower.
    • Claude Sonnet 4.6: The “heavy lifter” for complex logic.

    Tracking my OpenRouter activity during the initial build.

    The Minimalist Toolkit#

    I’m still a terminal guy at heart. There’s something nostalgic about working in the CLI that I don’t get in my day-to-day work. My editor of choice remains the “good old” Micro, and I use LazyGit for my GitHub workflow. The integration is seamless: I push a change, and the blog is updated via GitHub Actions in seconds.

    The “Connected Boxes” Moment#

    I did run into some initial compatibility issues between Hugo and the theme, but they were quickly resolved by using OpenCode to scan the folder structure. It was a perfect example of the “connected boxes” philosophy—when your tools talk to each other, even setup hurdles become easy to clear.

    It’s good to be home.
